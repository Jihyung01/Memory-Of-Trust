# S2-D: 대화 종료 감지 개선

## 목표
AI 응답에 대화 종료 시그널을 포함시켜, 자동 마이크 활성화를 끝내고 자연스럽게 대화를 마무리한다.

## 배경
현재 AI 말이 끝나면 무조건 자동 마이크 ON → 무음 6초 후 녹음 종료 → STT가 빈 결과 반환 → idle. 이 과정에서 불필요한 Whisper API 호출이 발생하고, 어르신 입장에서 "끝났는데 왜 또 듣고 있지?" 느낌을 줄 수 있다.

## 실행 계획

### 1. LLM 응답에 종료 플래그 추가
**파일**: `app/api/llm/respond/route.ts`

- 시스템 프롬프트에 추가 지시:
  ```
  대화를 마무리할 때 응답 마지막에 [END] 태그를 붙여주세요.
  예: "좋은 이야기 감사합니다. 다음에 또 들려주세요! [END]"
  어르신이 계속 이야기하고 싶어 하면 [END]를 붙이지 마세요.
  ```
- 응답에서 `[END]` 태그를 파싱하여 `should_end: boolean` 필드로 반환
- `response_text`에서 `[END]` 태그는 제거하여 TTS에 전달

### 2. 프론트 대화 종료 처리
**파일**: `app/(device)/device/[deviceId]/components/DevicePageClient.tsx`

- `llmData` 응답에 `should_end` 추가
- `should_end === true`이면:
  - TTS 재생 후 `audio.onended`에서 `autoStartRecording()` 대신 `setPhase("idle")`
  - 버블 텍스트에 종료 메시지 유지
  - `turnCount` 리셋

### 3. 턴 수 기반 종료 가드레일
- `turnCount >= 10`이면 LLM에 "자연스럽게 대화를 마무리해주세요" 추가 지시
- 토큰 비용 방지 + 어르신 피로 방지

## 주의사항
- `[END]` 태그가 TTS 음성에 포함되면 안 됨 — 반드시 제거 후 TTS 전달
- LLM이 `[END]`를 100% 정확히 붙이지 않을 수 있음 → 턴 수 가드레일이 백업
- 어르신이 "더 이야기할래"라고 하면 LLM이 알아서 [END] 안 붙이도록 프롬프트 설계

## 검증
1. 5턴 이상 대화 → AI가 자연스럽게 마무리 → 마이크 자동 종료
2. 어르신이 계속 이야기하면 → 대화 지속
3. 10턴 가드레일 → 강제 마무리 확인
4. Whisper 불필요 호출 횟수 감소 확인
