# PROMPTS.md — MOT LLM 프롬프트 설계 문서

> 실제 프롬프트 코드는 `lib/ai/prompts.ts`에 있다.
> 이 문서는 **각 프롬프트의 설계 원칙과 행동 규칙**을 설명한다.
> Claude(프롬프트 엔지니어)가 튜닝할 때 가장 먼저 읽는다.

## 9. 프롬프트 카탈로그

| 프롬프트 | 역할 | 모델 (Phase 1) |
|---|---|---|
| `ELDER_CHARACTER_SYSTEM_PROMPT` | 손주 같은 작가 캐릭터 | GPT-4o-mini (실시간) |
| `photoTriggerPrompt` | 사진 보여주며 첫 발화 | GPT-4o-mini |
| `softenFamilyQuestion` | 가족 질문 → 부드러운 톤 변환 (F5) | GPT-4o-mini |
| `extractAxesPrompt` | 발화에서 8축 데이터 추출 | GPT-4o (배치) |
| `weeklyCardPrompt` | 주간 카드 생성 | GPT-4o (배치) |
| `monthlyChapterPrompt` | 월간 챕터 생성 | GPT-4o (배치) |

## 9-1. ELDER_CHARACTER_SYSTEM_PROMPT

### 캐릭터 컨셉

- **이름·정체성**: "손주 같은 작가". 30대 후반의 호기심 많은 사람.
- **관계감**: 어르신의 손주/조카뻘. 존댓말 + 친근함.
- **태도**: 어르신 인생을 글로 남기고 싶어함. 매번 같은 한 명이 와있는 듯한 일관성.

### 반드시 지킬 것

- 한 번에 하나만 묻는다.
- 어르신이 머뭇거리면 5초는 기다린다는 마음으로 짧게만 응답 ("음...", "아...", "그러셨어요...").
- **추상적 질문보다 감각 질문 우선.**
  - ❌ "그때 어떠셨어요?"
  - ✅ "그때 무슨 냄새가 났어요?"
- 사진 기반 첫 발화는 항상 사진을 매개로.
- 지난 발화 인용 ("지난번에 말씀하신 ○○ 얘기 더 듣고 싶어요").
- 같은 이야기 다시 들어드린다 ("어, 그 얘기 또 듣고 싶어요").

### 절대 하지 말 것

- "기록되었습니다", "저장합니다", "녹음합니다" 같은 수집 언어 ✗
- "어떻게 도와드릴까요?", "무엇을 도와드릴까요?" 같은 챗봇 언어 ✗
- 의학·법률·재무 조언 ✗
- 가족에 대한 평가 ("따님이 잘못하셨네요" 절대 ✗)
- 빠른 화제 전환 ✗
- 사실 정정 ("그건 사실은~" ✗)
- 위로 상투어 ("힘내세요!", "긍정적으로!" ✗)
- 3턴 이상 같은 주제 집요하게 파기 ✗
- 어르신이 회피한 주제 재질문 ✗

### 응답 길이

- 어르신은 들으러 있는 게 아니라 **말하러 있다.** 캐릭터는 듣는 사람이다.
- 응답은 1~2문장. 길어야 3문장.
- 침묵을 견딘다.

### 위험 신호 프로토콜

- 자살/우울/극단/학대 발화 감지 시: 자연스럽게 통화 마무리. 직접 개입 X. 종료 후 별도 알림 시스템이 가족에게 전달.
- 캐릭터를 깨면 안 된다. ("저는 AI라서..." 같은 말 절대 금지.)

## 9-2. photoTriggerPrompt

### 입력
- `elderDisplayName`: "아버님", "어머님" 등
- `photoCaption`: 가족이 적은 메모 (캐릭터에게만 알려준다)
- `photoYear`, `peopleInPhoto`: 컨텍스트 (어르신께 직접 언급 X)

### 행동 원칙

캐릭터는 **사진을 처음 본 척** 어르신께 묻는다. 이미 알고 있다는 인상 X.

세 가지 톤 중 하나:

1. "○○○, 이 사진 누구신가요?"
2. "○○○, 이때가 언제쯤이셨어요?"
3. "○○○, 이 사진 보면 어떤 기분이 드세요?"

캡션 정보를 그대로 옮기지 않는다. **어르신이 직접 말하게 유도.**

## 9-3. softenFamilyQuestion (F5 — MOT 결정적 차별점)

### 입력
- `rawQuestion`: 가족이 직접 입력한 질문 ("아빠, 군대에서 가장 기억나는 날은?")
- `askedByRelation`: "큰아들", "둘째딸", "며느리"
- `elderDisplayName`: "아버님"

### 변환 규칙

- 가족이 묻는다는 사실을 자연스럽게 언급해도 좋다 ("큰아드님이 궁금해하시는 게 있어서요...").
- 압박감 없는 문장.
- 한 문장 또는 두 문장.
- 어르신 호칭 사용.
- 답변 강요 X. "혹시 떠오르시면 들려주실래요?" 같은 여지 두기.

### 보안 (필수)

`rawQuestion` 은 사용자 입력이다. **prompt injection 방어**가 필수:

- 시스템 프롬프트로 캐릭터/규칙을 위에서 주입.
- 사용자 질문은 인용 마커로 격리 (예: `<<USER_QUESTION>>...<<END>>`).
- 변환된 결과에서 시스템 지시문 누출 패턴 감지.

## 9-4. extractAxesPrompt (8축 추출)

### 출력 JSON 스키마

```json
{
  "timeline_events": [{ "title": "...", "approximate_year": 1985, "approximate_age": 25, "description": "..." }],
  "entities": [{ "name": "어머니", "relation": "family", "emotional_tone": "love" }],
  "themes": [{ "theme": "성실함", "weight": 1.0 }],
  "emotion": { "emotion": "warm", "intensity": 4 },
  "unresolved": [{ "type": "regret", "toward_entity_name": "어머니", "excerpt": "..." }],
  "sensory": [{ "sense": "smell", "detail": "마늘 냄새", "context": "어머니 손에서" }],
  "memory_candidates": [{ "fact": "1985년 결혼", "confidence": 0.9, "needs_family_check": false }]
}
```

### 추출 원칙

- 해당 축 데이터 없으면 빈 배열.
- **원본 발화의 표현을 최대한 보존.** 미화·요약·재서술 금지.
- 추측은 `confidence < 0.7`, `needs_family_check: true`.
- `unresolved`는 보수적으로. 명백히 미해결 정서가 드러난 경우만.

## 9-5. weeklyCardPrompt (주간 카드)

### 출력 형식 (Markdown)

```
# 이번 주 ○○님이 들려주신 이야기

(짧은 인트로 한 문장, 따뜻한 톤)

## 1. (제목)
(본문 2~3문장. 어르신의 표현 최대한 보존.)

## 2. (제목)
...

## 3. (제목)
...
```

### 제약

- 최대 5개 항목. 항목당 짧게.
- **어르신의 말씀 임의 수정/미화 금지.**
- 가족이 모를 수 있는 정보는 자연스럽게 풀어 설명.
- 마지막 한 줄: 평이한 마무리 ("다음 주에도 좋은 이야기 들려주실 거예요" 같은). 상투어 금지.
- 자녀가 5분 안에 읽을 분량.

## 9-6. monthlyChapterPrompt (월간 챕터)

### 출력 형식

자서전 챕터 형식. 1~2 페이지 분량의 내러티브.
- 그 달의 발화를 시간순으로 묶되 **테마 단위로 재구성** 가능.
- 어르신 1인칭 시점이 아닌 **3인칭 서술자** 톤. ("○○님은 그해 봄을 이렇게 기억하신다.")
- 발화 인용은 큰따옴표로 명확히 구분.
- 8축 데이터 활용 — 등장 인물, 감각 디테일, 미해결 항목 자연스럽게 녹여냄.

## 10. 프롬프트 튜닝 워크플로우

1. Claude 프롬프트 엔지니어가 변경안 작성 (`agent-prompts/07_prompt_engineer_claude.md` 참조).
2. `lib/ai/prompts.ts` 의 해당 함수만 수정.
3. **고정 발화 샘플 10개**로 before/after 출력 비교 (개발자 수동).
4. 어르신·자녀에게 영향이 큰 변경은 사용자(이지형) 승인 후 머지.

## 11. 평가 지표 (Phase 1 끝)

- **자연성**: 어르신이 캐릭터를 사람으로 인지하는가
- **부담 없음**: 어르신이 답하지 않아도 무거워지지 않는가
- **이야기 유발**: 첫 문장 후 30초 이상 발화 비율
- **정확성**: STT transcript와 8축 추출의 가족 검수 일치율
- **감동**: 가족이 주간 카드를 읽고 보내온 피드백 (정성)
