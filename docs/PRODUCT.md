## 1. 본질

> **"인간은 죽기 전에 자신의 가치를 남기고 가고 싶어한다. 기억되고 싶어한다."**

MOT는 **자서전 서비스가 아니다.** 인간의 보존 욕구를 충족시키는 서비스다.
자서전은 1차 산출물일 뿐이고, 같은 데이터에서 멀티 아웃풋이 나온다 — 편지·미래편지·사과집·가치관집·추모영상.

---

## 1-1. 타겟

| 그룹 | 동인 | 비고 |
|---|---|---|
| 시한부 환자 | 긴급성 | 직접적 동인 |
| 70~90대 일반 어르신 | 일상화된 무의식적 욕구 | MVP 타겟 |
| 50~60대 자녀 | 부모를 잃기 전의 죄책감/효심 | **실제 결제 동력** |
| 자산가 | 레거시, 상속 정리 | Phase 3+ |
| 독거 노인 | 외로움, "내가 있었다는 흔적" | Phase 2+ |
| 입양인/이산가족 | 정체성의 빈 칸 | Phase 4+ |

**핵심 구조: 사용자는 어르신, 결제자는 자녀. 90% 자녀 결제, 10% 본인.**

---

## 1-2. 차별점 — 시간의 자산화

| 항목 | 기존 자서전 대필 | MOT |
|---|---|---|
| 인터뷰 횟수 | 5~10회 | 매일 누적 (50회+) |
| 누적 발화량 | 10시간 내외 | 50시간+ |
| 비용 | 500~3000만원 | 9.9~30만원/월 + 자서전 별도 |
| 산출물 | 책 1권 | 책 + 편지 + 영상 + 추모물 (멀티) |
| 핵심 자원 | 작가 1명의 시간 | **시간이 누적된 데이터** |

→ 시간의 자산화가 MOT의 해자(moat). **한 번의 인터뷰가 아니라, 매일 거실에서 쌓이는 데이터는 인간 작가로는 만들 수 없다.**

## 1-3. MVP 한 줄 정의

> **MOT 기억 라디오 액자** — 시판 안드로이드 태블릿을 라디오형 거치대에 넣고, PWA(추후 Expo 래퍼)로 키오스크 모드로 잠궈 운영하는 거치형 AI 기억 수집 장치.

## 2. MVP 5대 핵심 기능 (Phase 1~2)

이 5개만 만든다. 그 이상은 욕심.

### F1. 가족 사진 업로드

가족이 자녀용 웹에서 사진을 업로드. **사진은 가장 강력한 기억 트리거.** 첫 대화는 항상 사진에서 시작.

### F2. 정해진 시간에 부드럽게 말 걸기

하루 3회 (오전 10:30, 오후 3:00, 저녁 7:00), ±15분 랜덤. 어르신이 답 안 해도 자연스럽게 음악/사진 슬라이드로 넘어감. **대답 압박 0.**

### F3. 원본 무조건 보존 (Immutable)

모든 발화는 `raw_utterances` 테이블에 append-only로 저장. 후속 처리는 별도 레이어(`memory_candidates`, `timeline_events`, `story_outputs`)에서 진행. **원본은 절대 수정/삭제 X.**

### F4. 가족에게 주간 카드 자동 전송

매주 일요일 저녁, 그 주의 발화를 5개 항목으로 요약한 "이번 주 ○○님이 들려준 이야기" 카드를 가족에게 카카오 알림톡 + 이메일로 전송. **결제 유지의 핵심.**

### F5. 가족 질문 업로드 ⭐ MOT 결정적 차별점

가족이 자녀 웹에서 직접 질문 입력. AI가 어르신의 페이스로 자연스럽게 변형해서 부드럽게 대신 물음.

> "AI가 묻는 게 아니라, 자식이 묻고 싶어한 걸 AI가 대신 부드럽게 전달."

## 3. 산출물 (Outputs)

### 정기 산출물

- **주간 카드** (자동, 매주 일요일) — F4
- **월간 챕터** (자동, 매월 마지막 주) — 인생 타임라인 1장 + 자서전 초안 챕터 1개

### 연간/요청 산출물

- 자서전 완성본 (1년 후 또는 별도 신청)
- 개인별 편지 (자녀/손주 각각)
- **미해결 정리집 (사과/감사)** ⭐ 정서 강도 최고
- 미래 편지 (시간 잠금)
- 가치관집, 레시피북, 가족 신화

### 추모용 (Phase 3+)

- 디지털 묘비 페이지
- 추모 영상 (본인 음성 + 사진)
- 본인 목소리 클로닝 편지 (XTTS-v2 활용, 다른 자서전 서비스 절대 못 만듦)

## 15. 리스크 / 미해결 결정

### 기술

- 어르신 사투리 STT 정확도 → Phase 1에서 30회 발화 후 평가
- 어르신 집 와이파이 환경 → Phase 2에서 LTE eSIM 옵션 검토
- 화면 손상 → 케이스 단단하게, AS 정책 필요

### 운영

- 1년 구독 유지율 → 월간 챕터 품질이 핵심 변수
- 디바이스 회수 (해지/사망) → 운영 SOP 작성 필요

### 법

- 음성/대화 녹음 동의 (본인 + 가족)
- 사후 데이터 권한 (가족 접근 범위)
- 개인정보보호법 (어르신 발화 = 민감정보)

### 즉시 결정 필요 (Sprint 0~1 중)

- [ ] 클로바보이스 voice_id 어떤 걸 캐릭터 음성으로 쓸지 (`vara` vs `nshin` vs `nara`)
- [ ] PWA 설치 강제 vs 브라우저 풀스크린만
- [ ] 디바이스 분실/도난 시 raw_utterances 보호 (디바이스 토큰 회전)
