## 0. 한 문장 정의

> **MOT는 사람이 죽고 나서 모든 것이 0으로 돌아갈 때, 그 사람의 기억·목소리·관계·감정·삶의 서사를 1로 남기는 AI 기억 인프라다.**

핵심 슬로건:
> 사람은 사라져도, 기억은 남는다.
> MOT는 그 기억을 책임지는 시스템이다.

## 1. 사업 정체성

### 1.1 본질
MOT는 자서전 앱이 아니다. **인간의 보존 욕구**를 충족시키는 기억 인프라다.

자서전은 1차 산출물일 뿐. 같은 데이터에서 멀티 아웃풋:
- 자서전 / 인생 연대기 / 가치관집 / 레시피북
- 가족 편지 / 미래 편지 / 사과·감사 정리집 ⭐
- 사후 메시지 / 추모 영상 / 디지털 묘비

### 1.2 출발 감정
사용자는 사업 카피로 끌리는 게 아니라 **두려움 + 욕망**으로 끌린다:
- "내가 죽으면 아무도 나를 기억 못 할 것 같다" (두려움)
- "내 손주가 어른이 됐을 때 나를 알았으면 좋겠다" (욕망)
- "어머니가 돌아가시기 전에 그분 인생을 정리해드리고 싶다" (효심 + 죄책감)

이 감정이 결제 동력이다.

### 1.3 차별점: 시간의 자산화
| 항목 | 기존 자서전 대필 | MOT |
|------|---------|-----|
| 인터뷰 횟수 | 5~10회 | **수백 회 (1년+)** |
| 누적 발화량 | 10시간 | **50~100시간+** |
| 비용 | 500~3,000만원 | **24~120만원/년** |
| 산출물 | 책 1권 | **주간 카드 + 월간 챕터 + 자서전 + 편지 + 영상** |
| 핵심 자원 | 작가의 시간 | **AI + 누적 데이터** |

인간 작가가 만들 수 없는 양·결의 데이터. **시간이 곧 자산.**

## 2. 3대 축 (Care / Memory / Trust)

| 축 | 의미 | 기능 | 수익 구조 |
|---|---|---|---|
| **Care** | 살아있는 동안 말 걸고 들어줌 | AI 대화, 정서 체크, 위험 신호 감지 | 월 구독 |
| **Memory** | 삶의 이야기를 구조화 | 자서전, 사진 인터뷰, 8축 데이터 | 데이터 락인, 패키지 판매 |
| **Trust** | 기억의 공개·보존·상속 관리 | 사후 메시지, 권한 관리, 기억 신탁 | **고마진 장기 수익** |

**Trust 축이 5년 후 진짜 돈이 되는 영역.**
- 보험사 제휴 (상속/유언 패키지)
- 법무법인 제휴 (디지털 유산 신탁)
- 장례식장 제휴 (추모 콘텐츠 자동 생성)

이 3축 중 **Care부터 시작**해서 Memory로 자산 누적, Trust로 고마진 잠금. 순서가 중요하다.

## 3. 타겟 고객

### 3.1 4가지 페르소나

#### A. 시한부 환자 (긴급성 최고)
- 결제 결정 빠름. 가격 민감도 낮음.
- 대상: 본인 또는 자녀
- 채널: 호스피스, 종합병원 협력
- 패키지: **Legacy + Book Package** 묶음

#### B. 70~90대 어르신 (지속적 시장)
- 일상화된 보존 욕구 (의식 X, 무의식적)
- 결제 결정자: **자녀 (40~60대)** ⭐
- 사용자와 결제자가 다름 → 가족 참여 구조 필수
- 패키지: **Family** 가장 적합

#### C. 자산가 / 고령자 (고가치 시장)
- 레거시, 상속 정리 욕구
- 결제력 ◎
- 채널: 프라이빗 뱅킹, 상속 전문 법무법인
- 패키지: **Legacy + Book Package**

#### D. 자기 기록을 남기고 싶은 일반인 (확장 시장)
- 30~50대 본인이 결제
- 부모 세대가 아닌 본인 인생 기록
- 채널: 콘텐츠 마케팅 (인스타/유튜브)
- 패키지: **Memory Start**

### 3.2 결제자 vs 사용자
| 페르소나 | 결제자 | 사용자 | 결제 동기 |
|---|---|---|---|
| 시한부 | 본인 또는 자녀 | 본인 | 마지막 정리 |
| 어르신 일반 | **자녀** | 부모 | 효심 + 죄책감 |
| 자산가 | 본인 | 본인 | 레거시, 상속 |
| 일반인 | 본인 | 본인 | 자기 기록 |

**핵심**: 어르신 일반 페르소나에서 사용자 ≠ 결제자. 가족 참여 구조가 결제 유지의 핵심.

## 4. 시장 포지셔닝 (HereAfter AI 벤치마킹)

### 4.1 HereAfter AI 분석
- **창업자**: James Vlahos (시한부 폐암 진단 받은 아버지로 "Dadbot" 만든 게 시작)
- **모델**: 인터뷰 답변 녹음 → 가족이 음성으로 질문 → 저장된 답변 retrieval
- **가격**: $3.99 / $5.99 / $7.99 (월)
- **본질**: 정적 retrieval (어르신이 미리 녹음한 답만 재생)

### 4.2 HereAfter에서 가져올 것
1. **AI 자동 인터뷰어** — 카테고리화된 질문 (어린 시절/가족/사랑/직업/후회/가치관)
2. **실제 목소리 저장** — 텍스트보다 강한 정서 자산
3. **사진과 기억 연결** — 사진은 가장 강력한 기억 트리거
4. **가족/친구 초대** — 결제 동력
5. **선물 패키지** — 어버이날/생일/은퇴/명절

### 4.3 MOT의 결정적 차별점
| | HereAfter | **MOT** |
|---|---|---|
| 응답 방식 | 정적 retrieval (녹음한 답만) | **동적 응답 (AI가 8축 기반 새 응답 생성)** |
| 새 정보 반영 | 어르신이 다시 녹음해야 함 | **즉시 반영 (RAG로 자동)** |
| 산출물 | 음성 아카이브만 | **자서전 + 편지 + 카드 + 영상 멀티** |
| 가족 질문 | 가능 (저장된 답 한정) | **AI가 가족 질문을 부드럽게 변환해서 어르신에게 전달** ⭐ |
| 한국어/한국 정서 | 영어 중심 | **한국어 어르신 화법 특화 (LoRA 학습)** |
| 톤 | 일반 챗봇 | **"손주 같은 작가" 캐릭터 일관성** |

### 4.4 윤리적 표현 차별화
HereAfter는 "돌아가신 분과 대화한다"는 표현 회피. MOT도 같은 원칙:

> **MOT 권장 표현**: "고인이 남긴 기록에 질문한다"
> "기억 아바타는 사용자가 남긴 실제 음성·사진·이야기를 바탕으로 관련 기억을 찾아주는 대화형 아카이브다."

**"돌아가신 분이 답한다"는 표현 절대 금지.**

## 5. 요금제 v1

### 5.1 핵심 원칙
> **대화 횟수를 제한하면 안 된다. 제한해야 하는 것은 고급 산출물과 처리 시간이다.**
> **기록은 넉넉하게, 가공은 요금제별로, 원본은 절대 잃지 않게.**

월 4회 인터뷰 같은 구조는 본질 파괴. "맛보기 기록 앱"이 됨.

### 5.2 절대 제한 X
1. 원본 기억 저장
2. 짧은 일상 대화 (Care 축)
3. 핵심 기억 노드 추출
4. 기본 전사 텍스트
5. 가족에게 남길 핵심 메시지 초안

### 5.3 제한 가능
1. 월간 고급 자서전 챕터 수
2. 고품질 TTS 낭독 파일 수
3. 사진 기반 인터뷰 업로드 수
4. 가족 계정 수
5. MP3/원본 전체 다운로드
6. 사후 예약 메시지 수
7. 사람 편집자 검수
8. 인쇄 책/PDF 제작

### 5.4 요금제 구조

| 요금제 | 월 가격 | 핵심 목적 | 처리 한도 | 고급 산출물 | 가족 기능 |
|---|---:|---|---|---|---|
| **Memory Start** | 19,900원 | 처음 시작하는 기억 기록 | 월 300분, 횟수 제한 X | 월 챕터 1개, 카드 10개 | 가족 1명 |
| **Family** | 49,000원 | 가족이 함께 부모의 삶 보존 | 월 900분, 횟수 제한 X | 주간 편지, 월 챕터 2개, 사진 100장 | 가족 5명, 질문함 |
| **Legacy** | 99,000원 | 시한부/고령자 고밀도 기록 | 월 1,800분, 횟수 제한 X | 사후 메시지, 전체 MP3, 분기 자서전 | 가족 10명, 권한 관리 |
| **Book Package** | 80~300만원 | 완성형 자서전/가족사 | 별도 인터뷰 + 기존 데이터 | 사람 편집 + PDF/인쇄책 | 가족 납품 |

### 5.5 가격 합리화
- **MOT는 HereAfter보다 5~10배 비싸다 ($3.99 → 19,900원)**
- 한국 시장 가격 민감도보다 **정서 강도 + 한국어 품질 + 자녀 결제 의지**가 더 결정적
- 자녀가 부모 1년 자서전에 24만원 (Memory Start) ~ 119만원 (Legacy)은 효심 가격대로 적정
- HereAfter는 영어/북미 중심. 한국 어르신엔 부적합.

## 7. 사업 모델 확장 (Trust 축)

### 7.1 패키지 판매 (단발성 고가 매출)
- **Book Package**: 자서전 출판 (80~300만원)
- **Memorial Package**: 사후 추모 영상/디지털 묘비 (200~500만원)
- **Legacy Letter Package**: 가족별 개인 편지 묶음 (100~300만원)

### 7.2 B2B 채널
1. **호스피스/종합병원**: 시한부 환자 대상 (Phase 2~3)
2. **요양원/실버타운**: 어르신 단체 가입 (Phase 3+)
3. **장례식장**: 사후 콘텐츠 자동 생성 (Phase 3+)
4. **상조회사**: 상조 패키지에 디지털 추모 포함 (Phase 4+)
5. **보험사**: 상속/유언 신탁 패키지 묶음 (Phase 4+)
6. **법무법인**: 디지털 유산 관리 (Phase 4+)

### 7.3 데이터 자산 가치
- 한국어 어르신 발화 데이터 = 다른 곳에 없는 자산
- 정서적 한국어 학습 데이터로 LoRA 모델 자산화
- **사용자가 늘수록 모델이 강해지는 해자(moat)**

## 8. MVP 우선순위

### 8.1 0차 MVP (검증 단계)
**검증 질문 3가지**:
1. 사람이 정말 AI에게 자기 삶을 말하는가?
2. AI가 정리해준 기억을 보고 감동하는가?
3. 가족이 돈을 낼 만하다고 느끼는가?

**필수 기능 7가지**:
1. AI 음성/텍스트 인터뷰
2. 원본 음성 저장
3. 전사 텍스트 저장
4. 기억 노드 추출 (8축)
5. 자서전 문단 생성
6. 가족 공유 링크
7. 사진 기반 기억 인터뷰

**플랫폼**: 웹 PWA (디바이스 X)

### 8.2 1차 차별화 (Family 패키지 정당화)
1. 오늘의 기억 카드 (가족에게 자동 전송)
2. 가족 질문함 (가족이 질문 → AI가 부드럽게 변환해서 어르신에게)
3. 기억 편지 (가족별 개인 편지 초안)
4. 월간 자서전 챕터
5. 원본 음성 보관함

### 8.3 2차 확장 (Legacy 패키지 정당화)
1. 기억 아바타 (저장된 기록에 가족이 질문)
2. 사후 메시지 예약 전달
3. Final Visitor 모드 (시한부 마지막 정리)
4. 디지털 기억관 (웹 페이지)
5. PDF/인쇄 자서전 패키지

### 8.4 3차 디바이스 (Phase 2~3)
- 7인치 안드로이드 태블릿 + 라디오 외형 케이스
- 키오스크 모드 (어르신은 앱으로 인지 X)
- 시계 + 사진 + 한 문장 + 마이크
- 정해진 시간 자동 호출

## 12. 윤리·법적 가이드라인

### 12.1 데이터 동의 (필수)
- 어르신 음성 녹음 동의
- 가족 데이터 접근 권한 동의
- 사후 데이터 처리 권한 동의 (가족이 어디까지 접근?)
- **학습 데이터 사용 동의** (개인 식별 정보 제거 후 모델 개선용)

### 12.2 사후 데이터 정책
- 어르신 사망 후 데이터는 누구 소유인가?
- 가족이 데이터 삭제 요청하면?
- 가족 간 분쟁 시 (한 자녀가 비공개 원함, 다른 자녀가 공개 원함)?
- → **가입 약관에 명시 필수**. 변호사 검토 필요.

### 12.3 표현 가이드라인
- "AI가 돌아가신 분의 답을 생성한다" → ✗ 금지
- "고인이 남긴 기록에 질문한다" → ◎ 권장
- "기억 아바타와 대화한다" → ✗ 금지 (디지털 영생 오해)
- "기억 아카이브에서 관련 이야기를 찾아준다" → ◎ 권장

### 12.4 위험 신호 대응
- 어르신 발화에서 자살/우울/극단/학대 감지 시
- AI는 통화 자연스럽게 마무리. **절대 직접 개입 X**
- 통화 종료 후 가족에게 별도 알림 (SMS/이메일)
- 1366 (여성긴급전화), 129 (보건복지콜센터) 등 공식 자원 안내

- 디바이스 회수 운영 (해지/사망 시)
- 가족 분쟁 (사후 데이터 권한)

### 법적 리스크
- 음성/대화 녹음 동의 미비
- 사후 데이터 권한 분쟁

### 정서 리스크
