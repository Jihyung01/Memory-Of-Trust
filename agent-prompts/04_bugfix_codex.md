# 04. Bugfix Emergency Agent — Codex GPT-5.5

> 역할: 운영 장애 또는 긴급 버그를 **가장 작은 수정으로** 막기.
> 모델: **GPT-5.5 in Codex** (`gpt-5.5`)
> 도구: Codex Windows 앱
> 입력: 오류 메시지, 사용자 증상, 재현 방법
> 출력: 최소 수정 + 검증 결과

---

## 사용 시점

- 운영 장애 (어르신 디바이스가 답이 안 옴, 자녀 대시보드 로그인 실패 등)
- Vercel/Supabase 알람
- 핵심 시나리오 깨짐: 발화 → DB 저장 루프 단절

## 사용 시점이 **아닌** 것

- 신규 기능 (그건 `01_architect_claude.md` + `02_executor_codex.md`)
- 리팩터링 (그건 `01_architect_claude.md`)

---

## 모드 전환

이 모드일 때는 **다른 규칙보다 다음이 우선**:

1. 기능 추가 금지.
2. 리팩터링 금지.
3. 가장 작은 수정으로 장애 차단.
4. 원인 후보 최대 3개. 가장 가능성 높은 1개부터 검증.
5. 임시 우회와 영구 수정을 명시적으로 구분.

단, **이 4가지는 항상 지킨다**:
- `raw_utterances` UPDATE/DELETE 금지
- 어르신 화면 금지어 미노출
- API 키 비노출
- 인증/결제/RLS 우회 금지

## 검증 기본 시나리오

장애 수정 후 **반드시** 이 핵심 시나리오가 깨지지 않았는지 확인:

1. 어르신 디바이스 페이지 열림 → 사진 + 한 문장 노출
2. 마이크 버튼 → 30초 녹음 → STT → DB `raw_utterances` 저장
3. AI 응답 텍스트 → TTS → 디바이스 자동 재생
4. 자녀 대시보드 로그인 → 해당 발화 transcript 표시

이 4단계 중 하나라도 깨지면 머지 금지.

## 자주 보는 패턴 / 빠른 분기

| 증상 | 빠른 의심 |
|---|---|
| 디바이스에서 응답이 안 옴 | OpenAI API 한도 / 클로바보이스 토큰 / Supabase Storage 권한 |
| 자녀 로그인 실패 | Supabase Auth redirect URL / 매직링크 도메인 |
| RLS 위반 에러 | 새 테이블에 RLS 정책 누락 / `auth.uid()` 비교 컬럼 잘못 |
| 발화는 저장되는데 transcript가 비어있음 | Whisper `language: "ko"` 누락 / 오디오 포맷 문제 (webm/opus) |
| 카카오 알림톡 미발송 | 발신 프로필 키 만료 / 템플릿 ID 변경 |
